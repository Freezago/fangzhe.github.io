聊聊QIGS的依赖库

发表于 2019-04-23

国内的桌面版GIS及其二次开发几乎是ArcGIS一统天下。研一时因为项目原因需要进行QGIS的二次开发，才真正了解到GIS!=ArcGIS。

受中文社区很多博客的影响，许多人，包括我自己在内，QGIS二次开发的起手式就是——编译源码。由于依赖库繁多，编译耗时而又容易出错，因此这是一项十分考验人品和耐性的工作。在编译工作上浪费了漫长的人生后才知道，编译源码并不是一个必需的工作。OSGeo4W已经为我们提供了QGIS二次开发所需的release版开发包。

不过这总归不是一件完全没有用处的事情。在编译时我们可能会好奇，编译前的准备工作中，那么多的依赖库到底是干嘛的呢？作为QGIS的的忠实用户，总该对他们有所了解。

以我手中的QGIS3.4.5源码为例，INSTALL文件是这么告诉我们的：

Required build dependencies:

  • Qt >= 5.3.0
  • Proj >= 4.4.x
  • GEOS >= 3.4
  • Sqlite3 >= 3.0.0
  • SpatiaLite >= 4.2.0
  • libspatialindex
  • GDAL/OGR >= 2.1
  • Qwt >= 5.0 & (< 6.1 with internal QwtPolar)
  • expat >= 1.95
  • QScintilla2
  • QCA
  • qtkeychain (>= 0.5)
  • libzip

总共12个必需的依赖库，在写毕业论文的时候为了凑字数曾经对这12个库做过简单介绍，每个库的介绍都是翻译自官方文档，不过当时为了避免查重时撞车做了一些简单的语序调整和内容取舍。今天就从毕业论文里摘下来，做一些相应补充，与大家分享一下。

PROJ.4

通用坐标转换软件，用于坐标参考系统（Coordinate Reference System，CRS）之间的相互转换，包括地图制图时的投影和大地测量中的坐标转换。

这里说的坐标参考系统包括大地坐标和投影坐标，QGIS使用PROJ.4处理坐标转换过程中的数学运算。QGIS提供了5000余种用于世界各地的坐标系统，不过并不总是能满足我们的需求，如果想添加自定义投影的话，就需要对PROJ.4的投影描述有一定了解。不用担心，非常简单，花2分钟看一下文档就可以。例如，假如我们想使用中国地图常用的兰伯特等角圆锥投影，中央子午线为105°E，两条平行纬线分别为25°N与47°N。根据PROJ.4的格式，该投影的定义为

打开QGIS->Settings->Custom Projections，创建相应投影即可。

GEOS

全称 Geometry Engine - Open Source，用于处理空间拓扑关系，是Java 下著名拓扑关系库 JTS的 C++ 实现，致力于实现 JTS 的全部功能。GEOS 支持 Point、 LineString、
Polygon、 MultiPoint、 MultiLineString、 MultiPolygon、 GeometryCollection 等7种几何样式，有多种拓扑关系判断和空间分析函数。开源协议为LGPL。

QGIS 使用 GEOS 进行空
间关系判断（如包含、相交等）和操作（如合并、缓冲区等）。

参考：

  1. PyQGIS cook book: Geomety handling（ 几何处理）
  2. QGIS C++ 文档：QgsGeometry类

SQLite

一个十分轻量级的嵌入式跨平台数据库引擎，运行速度极快，
无需配置服务，可以十分方便地集成到其他应用程序。SQLite 在商业和开源领域均有众多应用，从操作系统（Android、IOS、Mac OS、Windows10），浏览器（Chrome、火狐、Safari），编程语言（PHP、Python）再到其他不可计数的应用程序，可以说，SQLite 是世界上部署最为广泛的数据库。

SpatiaLite

支持地理空间数据的 SQLite 扩展，支持将外部文件挂载为虚拟表并与 SQLite 连接。

libspatialindex

空间索引库，支持范围查询、位置查询、k-最邻近查询
等查询算法，易于部署和运行；提供空间数据插入、更新及删除的接口；提供各种定制功能，如分割算法、最小扇出系数等；支持内部和外部内存结构，实现索引的持久性。

GDAL/OGR

全称地理空间抽象数据库（Geospatial Data Abstraction Library，GDAL/OGR）是一个用于地理空间数据格式转换与处理跨平台库。GDAL
支持 140 余种栅格格式，OGR 支持 80 余种矢量格式。GDAL/OGR 为其支持的
数据格式提供抽象模型以供用户，开发者或其他应用程序调用，是当前应用最为
广泛的地理数据访问引擎。QGIS 使用 GDAL 库进行栅格数据访问，使用 OGR
库进行矢量数据访问。

Qwt

Qt Widgets for Technical Applications。基于 Qt 的 GUI 组件
库，主要用于绘制 2D 统计图，包括曲线图、频谱图、直方图等，此外也提供仪
表盘、罗盘、旋钮、轮子、滑动条、钟表等绘图元素的接口。QGIS 使用 Qwt 绘
制栅格影像的统计直方图。

Expat

使用 C 语言编写，是一个面向流的 XML 解析器库。作为最早
的开源 XML 解析器库，Expat 被许多开源项目采用，包括 Apache HTTP Server，
Mozilla 浏览器，Perl，Python 和 PHP 等。

QScintilla2

Scintilla 是一个用于开发代码编辑器的组件库，除基础的
文本编辑功能外，还有强大的代码编辑与调试功能，包括语法高亮、错误提示、
自动补全及调用提示，支持页边缘的端点和行号标记，提供多种字体样式选择 。
QScintilla 即是 Scintilla 的 Qt 实现。

QCA

全称 Qt Cryptographic Architecture，提供了一套简单的跨平台
加密接口，帮助开发者轻松地实现对称加解密、非对称加解密与及数字证书等功
能。

QtKeychain

用于存储密码或其他加密数据的 API。在 Windows 平台
上，QtKeychain 使用 Windows 底层 API 加密用户的密码和其他登录凭据，然后
使用 QSettings 保存加密数据。

libzip

用于操作压缩文件的 C 语言库，支持压缩文件的创建、读取和修改。
